### 목요일 6/4 — WHERE 심화 `+15분`

```sql
-- 비교/논리 연산자
SELECT * FROM posts WHERE likes >= 100 AND reach > 800;
SELECT * FROM posts WHERE type = '릴스' OR type = '카드뉴스';
SELECT * FROM posts WHERE type != '이미지';

-- 정렬
SELECT date, likes FROM posts ORDER BY likes DESC;
SELECT date, likes FROM posts ORDER BY likes DESC LIMIT 3;
```

---

### 금요일 6/5 — BETWEEN / LIKE / IN / NULL `+15분`

```sql
-- 범위 조건
SELECT * FROM posts WHERE likes BETWEEN 80 AND 150;
SELECT * FROM posts WHERE date BETWEEN '2026-05-01' AND '2026-05-31';

-- 패턴 매칭 (ADP 자주 출제)
SELECT * FROM posts WHERE caption LIKE '%마케팅%';
SELECT * FROM posts WHERE type IN ('릴스', '카드뉴스');

-- NULL 처리
SELECT * FROM posts WHERE caption IS NOT NULL;
```

## 2주차 (6/8 ~ 6/12)

### 월요일 6/8 — 집계 함수 `+15분`

```sql
-- 집계 함수 (ADP 핵심)
SELECT COUNT(*) FROM posts;
SELECT AVG(likes) FROM posts;
SELECT MAX(likes), MIN(likes), SUM(likes) FROM posts;

-- 별칭(alias)
SELECT AVG(likes) AS 평균좋아요, COUNT(*) AS 게시물수 FROM posts;
```

---

### 화요일 6/9 — GROUP BY `+15분`

```sql
-- GROUP BY (ADP 단골 문제)
SELECT type, COUNT(*) AS 게시물수
FROM posts
GROUP BY type;

SELECT type, AVG(likes) AS 평균좋아요
FROM posts
GROUP BY type
ORDER BY 평균좋아요 DESC;
```

---

### 수요일 6/10 — HAVING `+15분`

```sql
-- HAVING: GROUP BY 결과에 조건 (WHERE와 차이 이해 중요)
SELECT type, AVG(likes) AS 평균좋아요
FROM posts
GROUP BY type
HAVING AVG(likes) >= 100;
```

> **WHERE vs HAVING 차이 — ADP 단골 개념**
> 
> - `WHERE`: 그룹화 **전** 필터 → 개별 행에 조건
> - `HAVING`: 그룹화 **후** 필터 → 집계 결과에 조건

---

### 목요일 6/11 — CASE WHEN `+15분`

```sql
-- CASE WHEN (ADP 자주 출제)
SELECT
  date,
  likes,
  CASE
    WHEN likes >= 150 THEN 'A등급'
    WHEN likes >= 100 THEN 'B등급'
    ELSE 'C등급'
  END AS 성과등급
FROM posts;
```

## 3주차 (6/15 ~ 6/19)

### 월요일 6/15 — INNER JOIN `+15분`

```sql
-- 테이블 구조
-- posts(id, date, type, likes, reach, account_id)
-- accounts(id, name, followers)

-- INNER JOIN: 양쪽 다 있는 것만
SELECT posts.date, posts.likes, accounts.name
FROM posts
INNER JOIN accounts ON posts.account_id = accounts.id;
```

---

### 화요일 6/16 — LEFT JOIN `+15분`

```sql
-- LEFT JOIN: 왼쪽 테이블 기준, 오른쪽 없으면 NULL
SELECT accounts.name, COUNT(posts.id) AS 게시물수
FROM accounts
LEFT JOIN posts ON accounts.id = posts.account_id
GROUP BY accounts.name;
```

> **ADP 포인트**: LEFT JOIN에서 NULL이 생기는 경우 이해 필수

---

### 수요일 6/17 — JOIN 심화 (다중 조건 + 집계) `+15분`

```sql
-- 여러 조건 JOIN + 집계
SELECT
  accounts.name,
  posts.type,
  AVG(posts.likes) AS 평균좋아요
FROM posts
INNER JOIN accounts ON posts.account_id = accounts.id
GROUP BY accounts.name, posts.type
ORDER BY 평균좋아요 DESC;
```

---

### 목요일 6/18 — 서브쿼리 입문 `+15분`

```sql
-- 서브쿼리 (ADP 단골)
-- 평균보다 좋아요가 많은 게시물
SELECT date, likes
FROM posts
WHERE likes > (SELECT AVG(likes) FROM posts);
```

---

### 금요일 6/19 — 서브쿼리 심화 `+15분`

```sql
-- IN + 서브쿼리
SELECT * FROM posts
WHERE type IN (
  SELECT type FROM posts
  GROUP BY type
  HAVING AVG(likes) >= 100
);

-- EXISTS 맛보기
SELECT * FROM accounts a
WHERE EXISTS (
  SELECT 1 FROM posts p WHERE p.account_id = a.id
);
```

## 4주차 (6/22 ~ 6/26)

### 월요일 6/22 — WITH (CTE) `+15분`

```sql
-- CTE: 복잡한 쿼리를 읽기 쉽게 (ADP 실기에서 유용)
WITH high_performers AS (
  SELECT type, AVG(likes) AS avg_likes
  FROM posts
  GROUP BY type
)
SELECT * FROM high_performers
WHERE avg_likes >= 100
ORDER BY avg_likes DESC;
```

---

### 화요일 6/23 — 윈도우 함수 맛보기 `+15분`

```sql
-- RANK, ROW_NUMBER (ADP 실기 출제 범위)
SELECT
  date,
  likes,
  RANK() OVER (ORDER BY likes DESC) AS 순위
FROM posts;

SELECT
  type,
  date,
  likes,
  AVG(likes) OVER (PARTITION BY type) AS 유형별평균
FROM posts;
```

---

### 수요일 6/24 ~ 금요일 6/26 — ADP 기출 스타일 문제 풀기 `각 +15분`

```sql
-- 기출 유형 예시
-- Q. 각 콘텐츠 유형별 게시물 수, 평균 좋아요, 최대 도달수를 구하고
--    평균 좋아요가 100 이상인 유형만 출력하시오 (내림차순)

SELECT
  type,
  COUNT(*) AS 게시물수,
  ROUND(AVG(likes), 1) AS 평균좋아요,
  MAX(reach) AS 최대도달수
FROM posts
GROUP BY type
HAVING AVG(likes) >= 100
ORDER BY 평균좋아요 DESC;
```
