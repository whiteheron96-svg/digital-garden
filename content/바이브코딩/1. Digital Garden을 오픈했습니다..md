
어제 옵시디언과 Github을 연결하면서, '옵시디언으로 어떤것까지 해볼 수 있을까?'라고 생각하던 찰나, Github pages를 통해 블로그 글을 쓸 수 있다는 것을 알게 되었네요. 

덕분에, 기존에 노션에 흩어져있던 저만의 성찰 글들, 그리고 최근에 데이터 분석과 코딩을 공부하게 되면서 배운 내용을 정리하고 싶은 것들을 모아 오전에 만들어 보았습니다. 

노션은 그동안 공유하기 위해 클릭 여러번 해서 공유용 url을 만드는 게 불편했는데, 지금은 글을 쓰면 바로 배포할 수 있게 되니 아웃풋을 공유하기에 좀 더 편리하지 않을까 생각이 들었습니다. 

좀 더 깊은 설계의 생각을 정리할 예정입니다.

## 1. 개념 정리

|도구|역할|
|---|---|
|**Obsidian**|마크다운 노트 작성 도구|
|**GitHub**|노트 백업 & 버전 관리|
|**Quartz**|마크다운 → 웹사이트 변환 엔진|
|**GitHub Pages**|무료 웹 호스팅|

### 전체 흐름

```
Obsidian에서 글 작성
        ↓
GitHub에 push
        ↓
Quartz가 마크다운을 HTML로 변환
        ↓
GitHub Pages에 자동 배포
        ↓
나만의 Digital Garden 공개 🌿
```

## 9. GitHub Pages 배포

### deploy.yml 생성

```bash
cd ~/Documents/quartz
mkdir -p .github/workflows
cat > .github/workflows/deploy.yml << 'EOF'
name: Deploy Quartz site to GitHub Pages

on:
  push:
    branches:
      - main

permissions:
  contents: read
  pages: write
  id-token: write

jobs:
  build:
    runs-on: ubuntu-22.04
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      - uses: actions/setup-node@v4
        with:
          node-version: 22
      - name: Install Dependencies
        run: npm ci
      - name: Build Quartz
        run: npx quartz build
      - uses: actions/upload-pages-artifact@v3
        with:
          path: public

  deploy:
    needs: build
    permissions:
      pages: write
      id-token: write
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
EOF
```

### GitHub Pages 설정

```
레포 → Settings → Pages → Source → GitHub Actions 선택
```

### Push

```bash
git remote set-url origin https://github.com/[아이디]/[레포이름].git
git add .
git commit -m "add deploy workflow"
git push origin HEAD:main
```

### 사이트 확인

```
https://[아이디].github.io/[레포이름]
```

## 10. 자주 발생하는 오류 & 해결방법

### ❌ 오류 1: repository not found

```
fatal: repository '...' not found
```

**원인**: 레포 주소가 잘못됨  
**해결**:

```bash
git remote remove origin
git remote add origin https://github.com/[정확한아이디]/[정확한레포이름].git
```

---

### ❌ 오류 2: push 거부 (fetch first)

```
Updates were rejected because the remote contains work that you do not have locally
```

**원인**: GitHub에 파일이 있는데 로컬과 충돌  
**해결**:

```bash
git pull origin main --allow-unrelated-histories
git config pull.rebase false
git push -u origin main
```

---

### ❌ 오류 3: npm 권한 오류

```
npm error code EACCES
```

**원인**: npm 캐시 폴더 권한 문제  
**해결**:

```bash
sudo chown -R $(id -u):$(id -g) ~/.npm
npm install
```

---

### ❌ 오류 4: Quartz v5 빌드 오류

```
Error: Could not resolve "../../.quartz/plugins"
```

**원인**: Quartz v5.0.0 버그  
**해결**: v4로 다운그레이드

```bash
git checkout v4
npm install
```

---

### ❌ 오류 5: GitHub Pages Private 레포 오류

```
Upgrade or make this repository public to enable Pages
```

**원인**: 레포가 Private으로 설정됨  
**해결**: Settings → Danger Zone → Make public

---

### ❌ 오류 6: Jekyll로 빌드됨

**원인**: GitHub Pages Source가 Branch로 설정되어 Jekyll이 실행됨  
**해결**: Settings → Pages → Source → **GitHub Actions** 로 변경

---

### ❌ 오류 7: No upstream-branch is set

**원인**: Obsidian Git 플러그인 remote 브랜치 설정 미완료  
**해결**:

```bash
git branch --set-upstream-to=origin/main main
```

또는 Obsidian에서 remote 입력창 뜨면 `origin/main` 선택
