
일단, 환경변수(.env)파일에 API키와 다른 중요한 것들을 추가해놓고, 
스크립트가 실행될 때 해당 .env 파일을 불러오면서 실행이 된다. 

그러면, 새로운 .env 파일 (즉 로컬에만 있고 github에 안올라가는)들은 
git ignore를 해야 한다. 

```
###git ignore 어떻게 해? 

echo ".env" >> .gitignore #환경변수 파일을 gitignore에 포함시킨다 

```


그리고 이미 파일이 github에 올라갔을 수 있다. 
그럴 때는 추적 중인 파일을 제외해야 한다. 
왜냐? 

이미 git이 추적중이면 계속 올라가기 때문. 
즉, 한 번 .env를 생성하면 처음에 gitignore에 추가해놓아야 하고, 
그게 아니면 git status 구문을 입력해서 올라가 있는지 확인 후,
추적중 파일을 제외해야 한다. 

```
###추적 중인지 확인하고 추적중이면 캐시 제거하기 

git status ###추적 중인지 확인. 결과창에 목록이 뜨면 현재 추적중인 것 

git rm --cached 파일명  ###해당 파일은 추적을 제외한다

git rm -r --cached node_modules ###폴더 전체를 제거 
```

그 다음에 커밋 & 푸쉬를 하면 된다 

```
###커밋 & 푸쉬 하기 (깃헙과 연동시키기)

git add . 

git commit -m "변경사항에 대한 코멘트"

git push
```

이렇게 하면 Git에 중요 정보(API 키, 비밀번호 등)이 노출되지 않는다.
