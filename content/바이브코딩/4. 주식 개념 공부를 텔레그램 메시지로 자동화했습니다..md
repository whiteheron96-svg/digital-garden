# 노션 DB → 텔레그램 자동 발송 시스템 요약

  

매일 정해진 시간에 **Notion DB에 저장해 둔 주식 퀴즈**를 꺼내 **Telegram으로 자동 발송**하는 시스템입니다. 이 문서 하나로 전체 구조와 셋업 방법을 이해할 수 있도록 정리했습니다.

  

---

  
## 1. 한눈에 보는 전체 흐름

  

```

[Notion 퀴즈 DB] → [Python 스크립트] → [Telegram 봇] → [내 채팅방/그룹]

(퀴즈 저장) (조건에 맞는 퀴즈 조회) (메시지 발송) (수신)

↑ │

└─ 발송 후 "발송완료"로 상태 변경 ─┘

```

  

매일 두 번 동작합니다.

  

1. **문제 발송** (오전 8시경): `발송 상태 = 대기`이고 `날짜`가 오늘/과거인 퀴즈를 찾아 문제만 발송 → 상태를 `발송완료`로 변경

2. **정답 카드 발송** (약 40분 뒤): 방금 `발송완료`된 퀴즈의 정답 + 해설 + 관련 종목 시세/기사 링크를 발송

  

자동 실행은 **GitHub Actions**(컴퓨터가 꺼져 있어도 동작) 또는 **macOS launchd**(내 맥에서 동작) 중 하나로 합니다.

  

---

  

## 2. 준비물

  

| 항목 | 발급처 | 용도 |

|------|--------|------|

| Notion API Key | Notion 내 통합(integration) 생성 | DB 읽기/쓰기 |

| Telegram Bot Token | `@BotFather` | 메시지 발송 봇 |

| Telegram Chat ID | 봇에게 메시지 보낸 뒤 조회 | 어디로 보낼지 |

| FMP API Key (선택) | financialmodelingprep.com | 미국 종목 PER 보강 |

  

### Notion DB 구성 (data source ID)

- `주식 개념 퀴즈 DB`: `여기에 data source ID를 넣으면 됩니다'

- `주식 테마 DB`: `여기에 data source ID를 넣으면 됩니다'

- `주식 업종 DB`: `여기에 data source ID를 넣으면 됩니다'

- `주식 종목 DB`: `여기에 data source ID를 넣으면 됩니다'

- `주식 이슈 DB`: `여기에 data source ID를 넣으면 됩니다'

  

> ⚠️ Notion 통합(integration)에 위 DB들의 **연결(Connection) 권한을 공유**해야 스크립트가 접근할 수 있습니다.

## 3. 텔레그램 봇 만들기 + Chat ID 구하기

  

### 봇 만들기

1. Telegram에서 `@BotFather` 검색

2. `/newbot` → 봇 이름·username 설정

3. 받은 토큰을 `TELEGRAM_BOT_TOKEN`에 저장

  

### Chat ID 구하기 (개인 채팅)

1. 만든 봇과 대화를 열고 아무 메시지나 전송

2. 아래 스크립트 실행 → 출력된 숫자가 chat ID

  

```bash

cd <프로젝트 경로>/econ

set -a; source stock_quiz_bot/.env; set +a

python3 stock_quiz_bot/get_telegram_chat_id.py

```

  

### Chat ID 구하기 (그룹 채팅) — 친구와 공유할 때

1. 새 그룹 만들고 봇을 멤버로 **초대**

2. 봇이 그룹 메시지를 읽도록 설정 (둘 중 하나)

- 봇을 그룹 **관리자(admin)** 로 지정, 또는

- `@BotFather` → `/setprivacy` → 봇 선택 → **Disable**

3. 그룹에 아무 메시지나 전송 (privacy를 안 껐다면 `@봇이름` 멘션해서 전송)

4. 위 `get_telegram_chat_id.py` 다시 실행 → 그룹 ID 확인

  

> 📌 **그룹 chat ID는 음수(-)** 입니다.

> - 일반 그룹: `-123456789`

> - 슈퍼그룹/채널: `-100xxxxxxxxxx`

> 부호 `-`까지 전부 포함해서 넣어야 합니다.

## 4. 환경 변수(.env) 설정

  

`config.example.env`를 복사해서 `.env`를 만들고 값을 채웁니다.

  

```bash

cp stock_quiz_bot/config.example.env stock_quiz_bot/.env

```

  

`.env` 주요 값:

  

```env

NOTION_API_KEY=secret_xxx

TELEGRAM_BOT_TOKEN=123456:telegram-bot-token

TELEGRAM_CHAT_ID=-1001234567890 # 그룹이면 음수

FMP_API_KEY= # 선택

NOTION_VERSION=2026-03-11

STOCK_QUIZ_TIMEZONE=Asia/Seoul

```

## 6. 자동 실행 (둘 중 택1)

  

### A. GitHub Actions (추천 — 맥이 꺼져 있어도 동작)

- 워크플로 파일: `.github/workflows/stock-quiz-telegram.yml`

- 스케줄(UTC 기준, KST는 +9시간):

- 문제 발송: 08:03 KST(`3 23 * * *`) + 백업 08:17 / 08:31

- 정답 카드: 08:43 KST(`43 23 * * *`) + 백업 08:57 / 09:11

- 백업이 여러 개인 이유: GitHub Actions 스케줄이 지연/누락될 수 있어 안전장치로 추가

- `workflow_dispatch`로 수동 실행도 가능

- **GitHub Secrets 등록 필요**: `Settings > Secrets and variables > Actions`

- `NOTION_API_KEY`, `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID`, (`FMP_API_KEY` 선택)

  

### B. macOS launchd (내 맥에서 동작)

- 포함된 설정:

- `com.stock-quiz.send-quiz`: 매일 08:00 문제 발송

- `com.stock-quiz.answer-card`: 매일 08:10 정답 카드 발송

- 등록:

```bash

launchctl load ~/Library/LaunchAgents/com.stock-quiz.telegram.plist

```

## 7. 퀴즈 데이터 운영

  

매일 보낼 퀴즈는 `주식 개념 퀴즈 DB`에 미리 만들어 둡니다. 주요 속성:

  

| 속성 | 설명 |

|------|------|

| 퀴즈 제목 | 예) 반도체 장비 업종은 어떤 테마와 연결될까? |

| 관련 업종 / 관련 테마 / 관련 종목 | relation |

| 문제 유형 | 객관식 / 단답형 / OX / 서술형 |

| 문제 | 실제 질문 |

| 보기 | 객관식 보기 |

| 정답 / 해설 | 정답 카드에서 발송 |

| 날짜 | 발송일 |

| 발송 상태 | `대기` → (발송 후) `발송완료` |

  

### 퀴즈 대량 생성

```bash

python3 stock_quiz_bot/generate_industry_theme_quizzes.py --days 30 --start-date 2026-06-05 --apply

```

- "업종은 아래 테마 중 어디와 가장 관련 깊을까?" 형식의 업종-테마 객관식 자동 생성

  

### 정답 카드 내용

`발송완료`된 최신 퀴즈를 찾아 다음을 발송:

- 정답, 해설, 관련 종목

- 티커/종목코드 있는 종목의 현재가·전일 종가·PER (Yahoo Finance / Stooq 기준)

- 관련 최신 기사 링크
