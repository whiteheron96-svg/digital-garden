# Quartz 블로그에 Google Analytics와 Supabase 댓글 기능 추가하기

최근 GitHub Pages와 Quartz로 운영하는 블로그에 방문자 통계와 댓글 기능을 추가했다. 생각보다 시행착오가 있었지만, 결과적으로는 Google Analytics와 Supabase를 이용해 원하는 기능을 구현할 수 있었다.

## 목표

추가하려던 기능은 두 가지였다.

1. 방문자 수 확인
    
2. 로그인 없이 댓글 작성
    

댓글 기능은 GitHub 계정이 없는 사람도 사용할 수 있어야 했기 때문에 Giscus 대신 Supabase를 선택했다.

---

# 1. Google Analytics 연결

먼저 Google Analytics 4 속성을 생성했다.

속성을 생성하면 아래와 같은 형태의 측정 ID가 발급된다.

```text
G-XXXXXXXXXX
```

Quartz 설정 파일에서 Analytics 설정을 수정했다.

```ts
analytics: {
  provider: "google",
  tagId: "G-XXXXXXXXXX",
},
```

또한 Quartz 기본 예제 URL을 실제 블로그 주소로 변경했다.

기존:

```ts
baseUrl: "quartz.jzhao.xyz"
```

수정:

```ts
baseUrl: "your-blog-url"
```

배포 후 Google Analytics 실시간 보고서에서 접속이 수집되는 것을 확인했다.

---

# 2. 댓글 시스템 설계

처음에는 Giscus를 고려했다.

장점:

- 설치가 간단함
    
- GitHub Discussions 활용
    

단점:

- GitHub 계정 필요
    

그래서 댓글 데이터를 직접 활용하기 위해 Supabase 기반으로 구현하기로 결정했다.

---

# 3. Supabase 댓글 테이블 생성

댓글은 게시글별로 분리해야 하므로 아래 구조를 사용했다.

```sql
create table public.comments (
  id bigint generated always as identity primary key,
  page_slug text not null,
  nickname text not null,
  content text not null,
  created_at timestamptz not null default now()
);
```

중요한 점은 게시글마다 테이블을 만드는 것이 아니라 댓글 테이블 하나만 사용한다는 것이다.

예시:

|page_slug|nickname|content|
|---|---|---|
|/money-school/|user1|좋은 글입니다|
|/money-school/|user2|감사합니다|
|/etf-guide/|user3|질문 있습니다|

---

# 4. RLS(Row Level Security) 설정

읽기 정책:

```sql
create policy "Anyone can read comments"
on public.comments
for select
to anon
using (true);
```

쓰기 정책:

```sql
create policy "Anyone can insert comments"
on public.comments
for insert
to anon
with check (true);
```

---

# 5. Quartz에 댓글 컴포넌트 추가

댓글 기능은 다음 구조로 구현했다.

- 현재 페이지 URL 읽기
    
- page_slug 생성
    
- 해당 slug의 댓글 조회
    
- 댓글 작성
    
- 댓글 저장 후 자동 새로고침
    

게시글 URL 예시:

```text
/posts/my-first-post/
```

댓글 저장 예시:

```text
page_slug = /posts/my-first-post/
```

---

# 6. 발생했던 문제

댓글창은 보였지만 댓글이 저장되지 않았다.

원인을 찾기 위해:

- Console 확인
    
- Network 탭 확인
    
- Supabase 요청 확인
    

을 진행했다.

결국 원인은 환경변수였다.

댓글 컴포넌트에서 아래 값을 읽도록 되어 있었는데,

```ts
process.env.PUBLIC_SUPABASE_URL
process.env.PUBLIC_SUPABASE_ANON_KEY
```

빌드 과정에서 값이 전달되지 않고 있었다.

Console에는 아래 메시지가 나타났다.

```text
Missing Supabase comments configuration
```

---

# 7. 해결

환경변수가 Quartz 빌드 과정에 전달되도록 수정한 뒤 댓글 등록 테스트를 진행했다.

이후에는:

```text
댓글 작성
↓
Supabase 저장
↓
댓글 목록 갱신
```

이 정상적으로 동작했다.

Network 탭에서도 Supabase 요청이 확인되었고, 댓글 데이터가 테이블에 저장되는 것을 확인했다.

현재 블로그는 다음 기능을 갖추게 되었다.

- Google Analytics 방문자 통계

예시:

```sql

from comments
group by page_slug
order by count(*) desc;
```

# 정리

정적 블로그라고 해서 방문자 통계나 댓글 기능을 포기할 필요는 없다.

Google Analytics와 Supabase를 활용하면 비교적 간단하게 블로그를 확장할 수 있으며, 댓글 데이터를 활용한 분석도 가능하다.

특히 Supabase 방식은 데이터를 직접 소유할 수 있다는 점에서 장기적으로 활용 가치가 높다.
